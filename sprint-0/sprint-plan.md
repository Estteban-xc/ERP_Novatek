# Plan del Sprint 0

## Duración del Sprint
- Inicio: 23 de febrero  
- Fin: 3 de marzo  

## Meta del Sprint
Establecer la estructura base del sistema ERP, preparar el entorno de desarrollo, organizar el repositorio del proyecto y definir los componentes iniciales necesarios para comenzar la implementación del sistema.

## Alcance del Sprint

Durante este Sprint se realizará:

- Organización del repositorio GitHub  
- Definición de estructura de carpetas del sistema  
- Configuración inicial del proyecto en Visual Studio  
- Preparación del esquema de base de datos MySQL  
- Documentación inicial del proyecto  
- Planificación de historias y tareas  

## Entregables del Sprint

- Repositorio estructurado profesionalmente  
- Carpeta Sprint-0 con planificación  
- Documentación técnica inicial  
- Esquema preliminar de base de datos  
- Configuración inicial del entorno  

## Evidencia de avance

El progreso del Sprint será evidenciado mediante:

- Commits en el repositorio GitHub  
- Actualización del README  
- Documentación almacenada en carpetas Docs y sprint-0  
- Actualización de GitHub Pages con la información del proyecto  

## Sprint Review

Fecha: 3 de marzo  

Se presentará:

- Organización del repositorio  
- Documentación generada  
- Estructura base del sistema  
- Preparación del entorno de desarrollo
